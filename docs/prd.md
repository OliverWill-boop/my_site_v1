# 需求文档

## 1. 应用概述

- **应用名称**：告白火山的个人主页
- **应用描述**：一个简约、清爽、带有科技感的个人主页，包含个人信息展示区与数字分身聊天区，主要面向老师和同学访问。

---

## 2. 用户与使用场景

- **目标用户**：老师、同学
- **核心使用场景**：
  - 访客通过主页快速了解「告白火山」的基本信息、当前在做的事情及兴趣方向
  - 访客通过数字分身聊天区向其提问，获取联系方式、作品信息或个人喜好等常见问题的回答

---

## 3. 页面结构与核心功能

### 3.1 页面总览

```
个人主页（单页）
├── 顶部导航栏
├── 个人信息区
│   ├── 头像
│   ├── 名字与一句话介绍
│   └── 个人信息展示区
└── 数字分身聊天区
```

### 3.2 顶部导航栏

- 展示站点名称或 Logo 文字「告白火山」
- 提供页面内锚点导航：「关于我」「数字分身」
- 固定在页面顶部，滚动时保持可见

### 3.3 个人信息区

**头像**
- 展示圆形头像占位图（默认使用首字母或默认头像样式）
- 头像尺寸适中，视觉突出

**名字与一句话介绍**
- 名字：告白火山
- 一句话介绍：正在积累个人项目，热衷于电子 DIY 的学生

**个人信息展示区**
- 当前在做：AI 应用、电子 DIY
- 兴趣方向：编程、运动、音乐、做有意思的小玩意儿
- 擅长与关注：具身智能、财经新闻、AI 应用
- 个人特点：可以把复杂的知识用人话讲出来
- 以标签或卡片形式呈现，信息层次清晰

### 3.4 数字分身聊天区

- 区域标题：「和我聊聊」或「数字分身」
- 聊天界面包含：
  - 消息展示区：展示对话历史（访客提问 + 数字分身回复）
  - 输入框：访客输入问题
  - 发送按钮：提交问题
- 数字分身预设知识库（用于生成回复的上下文信息）：
  - 身份：学生
  - 最近在做：积累个人项目
  - 擅长方向：具身智能、财经新闻、AI 应用
  - 高频问题预设回答参考：
    - 联系方式：由数字分身引导访客留言或说明获取方式
    - 当前作品：介绍正在积累的个人项目方向
    - 个人喜好：编程、运动、音乐、做有意思的小玩意儿
- 数字分身通过调用大语言模型 API 实现对话能力，基于上述预设知识库进行回复
- 对话记录仅在当前会话内保留，刷新后清空，无需持久化存储

---

## 4. 业务规则与逻辑

- 页面为单页应用，所有内容在同一页面内通过滚动访问，导航栏锚点定位至对应区域
- 数字分身聊天区：
  - 输入框为空时，发送按钮不可点击
  - 发送后输入框自动清空，消息展示区自动滚动至最新消息
  - 数字分身回复期间展示加载状态（如省略号动画或「思考中...」提示）
  - 单次输入字符上限：500 字

---

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 数字分身 API 请求失败 | 展示友好提示：「暂时无法回复，请稍后再试」 |
| 输入内容超过 500 字 | 输入框提示字数限制，超出部分不可继续输入 |
| 网络断开 | 发送失败时提示「网络异常，请检查连接后重试」 |

---

## 6. 视觉与交互规范

- **整体风格**：简约、清爽、带有科技感，不过度装饰
- **主色调**：深蓝色（主色）+ 白色（背景/辅助色）
- **移动端适配**：页面需适配手机屏幕，布局在小屏设备上保持可读性与可操作性
- **信息层次**：通过字号、颜色深浅、间距区分主次信息，避免视觉混乱

---

## 7. 验收标准

- 页面可正常访问，顶部导航锚点跳转功能正常
- 头像、名字、一句话介绍、个人信息区内容完整展示
- 数字分身聊天区可正常输入、发送，并展示回复内容
- 数字分身回复内容基于预设知识库，符合「告白火山」的身份设定
- 页面在手机端可正常浏览，布局无错位
- 整体视觉风格符合深蓝色 + 白色、简约科技感要求

---
